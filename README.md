# 🎓 Интерактивный портал самостоятельной работы студентов

**Дипломный проект** | Node.js + Express + SQLite + Anthropic AI

---

## 📋 Описание проекта

Веб-портал для организации самостоятельной работы студентов колледжа.
Студенты получают задания, загружают решения и получают автоматическую проверку от ИИ-ассистента.

## 🛠 Технологический стек

| Слой | Технология | Назначение |
|------|-----------|------------|
| Сервер | Node.js + Express | Обработка HTTP-запросов, REST API |
| База данных | SQLite (better-sqlite3) | Хранение данных пользователей, заданий, ответов |
| Авторизация | JWT (jsonwebtoken) | Аутентификация без сессий на сервере |
| Пароли | bcryptjs | Безопасное хэширование паролей |
| Файлы | Multer | Загрузка файлов от студентов |
| ИИ | Anthropic Claude API | Автоматическая проверка работ |
| Фронтенд | HTML + CSS + JS | Интерфейс без фреймворков |

## ⚙️ Установка и запуск

### 1. Установить зависимости
```bash
npm install
```

### 2. Настроить переменные окружения
```bash
cp .env.example .env
# Отредактируйте .env и вставьте ваш API-ключ Anthropic
```

### 3. Запустить сервер
```bash
node server.js
```

### 4. Открыть в браузере
```
http://localhost:3000
```

## 📡 API-эндпоинты

### Авторизация
| Метод | URL | Описание |
|-------|-----|---------|
| POST | /api/auth/register | Регистрация |
| POST | /api/auth/login | Вход в систему |
| GET | /api/auth/me | Данные текущего пользователя |

|-------|-----|---------|

## 🔐 Как работает авторизация (JWT)

```
1. Пользователь вводит email + пароль
2. Сервер проверяет пароль (bcrypt.compare)
3. Сервер создаёт JWT-токен (содержит id, role)
4. Браузер сохраняет токен в localStorage
5. Каждый запрос к API: Authorization: Bearer <токен>
6. Middleware проверяет токен → разрешает доступ
```

---

## 🤖 Как работает ИИ-проверка

```
1. Студент отправляет решение
2. Сервер формирует запрос: [задание + ответ студента]
3. Запрос отправляется в Anthropic Claude API
4. ИИ анализирует: что правильно, что ошибочно, советы
5. Ответ ИИ сохраняется в БД вместе с решением
6. Студент видит подробный отзыв сразу после сдачи
```

## 👨‍💼 Функционал по ролям

### Студент
- Просмотр списка заданий
- Сдача решений (текст + файл)
- Получение автоматической ИИ-проверки
- Просмотр оценок и отзывов
- Календарь дедлайнов
- Вопросы к ИИ-ассистенту

### Преподаватель
- Создание заданий (название, условие, предмет, дедлайн)
- Просмотр всех ответов студентов
- Чтение ИИ-отзывов по каждой работе
- Выставление финальной оценки
- Управление заданиями (удаление)
